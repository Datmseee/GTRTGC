This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

This is the **SPLRT Fleet Management Dashboard** for the Singapore RailTech Grand Challenge (SGRTGC) 2026. It tracks mileage and preventive maintenance (PM) for 59 trains on the Sengkang-Punggol LRT network.

## Running the Dashboard

No build tools required. Open `First_Dashboard.html` directly in any modern browser:

```bash
# Simple local server (avoids font CORS issues)
python3 -m http.server 8080
# then open http://localhost:8080/First_Dashboard.html
```

## Architecture

The entire frontend is a **single self-contained HTML file** (`First_Dashboard.html`) with inline CSS and JS — no frameworks, no bundler, no external JS dependencies.

### Key JS Data Structures

- **`STATIONS`** — the single source of truth for all station positions. Object keyed by loop ID (`'SK-West'`, `'SK-East'`, `'PG-West'`, `'PG-East'`), each value is an ordered array of `{name, x, y, hub?}` objects. Logical canvas coordinates in a `870×450` space.
- **`LOOPS`** — derived from `STATIONS` via `buildLoop()`. Each loop is pre-computed as ~400 interpolated points using Catmull-Rom spline. Used for both track rendering and train position animation.
- **`DEPOT`** — single `{x, y, name}` object for MMB Depot position.
- **`fleet`** — array of 59 train objects generated by `makeFleet()`. Each train has `{id, type, mileage, status, loop, pos, speed, ...}`.

### Rendering Pipeline

Canvas (`#mapCanvas`) renders at `devicePixelRatio` scale. Coordinate transforms:
- `mapXY(lx, ly)` — logical → CSS pixel (aspect-ratio-preserving with centering offset)
- `toCanvas(lx, ly)` — logical → physical canvas pixel (multiplied by DPR)

The `requestAnimationFrame` loop calls `drawFrame()` which: clears canvas → draws tracks → draws stations → draws depot → draws trains → updates sidebar.

### PM Threshold Logic

`getPMStatus(mileage)` computes the nearest upcoming PM cycle. Five cycles: `[2000, 13000, 40000, 120000, 360000]` km. "Soon" = within 8% of cycle length. "Overdue" = within 2% past a threshold.

### Layout

Three-panel layout: fixed `header` (52px) → `main` (flex row: map canvas + 310px side panel) → fixed `alert-strip` (38px footer ticker).

Side panel has three tabs: Fleet list, Alerts, Detail. Tab switching via `switchTab()`.

## What to Edit and Where

| Task | What to change |
|------|---------------|
| Fix station positions | `STATIONS` object (~line 345) — edit `x`/`y` values |
| Change track colors | CSS vars `--track-sk` and `--track-pg` in `:root` |
| Change PM thresholds | `PM_CYCLES` array (~line 418) |
| Add/remove trains | `makeFleet()` function |
| Depot position | `DEPOT` object (~line 384) |
| Depot connector line | `drawDepot()` function — currently connects to `Ranggung` station |

## Correct Station Coordinates

The briefing (`CLAUDE_CODE_BRIEFING.md` Section 3) contains the corrected geographic x/y coordinates for all stations in the `870×450` logical canvas space, including hub positions for Sengkang (~x=200, y=260) and Punggol (~x=580, y=260), and all four loops.

## Constraints

- **Do not split into multiple files** — must remain a single HTML file
- **Do not change** the dark theme palette, font choices (IBM Plex Mono + IBM Plex Sans Condensed), sidebar layout, detail panel design, train animation logic, or PM threshold values
- CSS variables for all colors are defined in `:root`; use them via `var(--name)`
- Font family references: `var(--mono)` for data/numbers, `var(--sans)` for UI text
